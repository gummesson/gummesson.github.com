Begrider

Begrider is a simple, fluid twelve column grid system with a 1140px fixed width (or technically 1137px, if you want to be pixel-precise). It works great in Chrome and Firefox and looks alright in Internet Explorer 9. It works in Opera too if you donâ€™t mind that some of the columns are a tiny bit off. The goal behind it was creating something that I could actually use and to make it as simple as possible. I think I did pretty good.

I know that there's about a thousand other grid systems available on the web and most of them are probably much more elaborate than mine and have more thought-out features. Like I said, I created Begrider as a learning tool of some sort and because I wanted to have a simple system that I could use.

The code

I think that the code is pretty straight forward, but since I'm a bit of a documentation lover, I'll write about it anyway.

CSS

The CSS consists of six important bits: .container, .grid, .row, [class="col-"], .col-, .last and the media queries.

.container

.container {
       -moz-box-sizing: border-box; 
    -webkit-box-sizing: border-box;
            box-sizing: border-box;

     width: 95%;
    height: 100%;
    margin: 0 auto;
}

The .container is essentially used for wrapping the whole screen in. The box-sizing: border-box; property makes it easy to add padding and borders without breaking the grid and columns. The width: 95%; and height: 100%; makes the .container stretch across nearly the whole screen while having fluid margins on the left and right side of it (it gets useful when the browser window gets smaller). margin: 0 auto; is used to align the .container to the center of the screen.

.grid

.grid {
    max-width: 1140px;
       margin: 0 auto;
}

The .grid class is the main wrapper for the whole grid layout. The default max-width is 1140px, but you could make it narrower, or wider, if you want to since Begrider is a fluid system. margin: 0 auto; is used for the same purpose as described about above.

.row

.row {
       clear: both;
    overflow: hidden;
}

The .row class is used as a wrapper for one specific row of columns. It enables you to have multiple rows of different columns in the .grid. The clear: both; clears the floating from the .col- classes and the overflow: hidden; makes the columns "stay in place" ny hiding any overflowing content.

[class="col-"]

[class^="col-"] {
      min-height: 1px;
    margin-right: 2.85%;
           float: left;
}

The [class="col-"], which is an attribute selector, is used to apply all the properties inside of it to all of the .col- classes. The min-height: 1px; enables the .col- classes to actually work without having any content inside of them. The margin-right: 2.85%; adds a so called gutter to the right side of every .col- and the float: left; makes the columns line up to the left.

The .col- classes contain the width of the columns. The widest one column is called .col-twelve and the narrowest .col-one because the former spans over twelve columns and latter spans over one.

.last

.last {
    margin-right: 0;
}

The .last class is used to remove the margin-right of the last .col- class in a .row. That means that if it's not applied the .row will break.

Media Queries

/* iPad/Tablet (Portrait) */

@media only screen and (max-width: 768px) {

    [class^="col-"] {
             width: 100%;
        min-height: 0;
             float: none;
    }

}

/* iPhone/Smartphone (Landscape) */

@media only screen and (max-width: 480px) {

    .container {
        width: 92.5%;
    }

}

The media queries are used to make the grid system adapt to different screen sizes. The first one is targeted at iPads/tablets used in portrait mode. The [class="col-"] is used to change the .col- classes so that they stop floating (which makes them stack on top of each other instead), remove the min-height so that the .col- classes without any content doesn't take up any unneccassary space and to stretch them out across the full width of the .grid instead. The second one is targeted at iPhones/smartphones when they're used in landscape mode. The width of the .container is change so that the .grid gets more room to breathe.

HTML

<div class="container">

    <div class="grid">

        <div class="row">

            <div class="col-one"></div>

                <div class="col-ten">Add content here...</div>

            <div class="col-one last"></div>

        </div><!--.row -->

    </grid><!--.grid -->

</div><!--.container -->

The markup is fairly simple too. Wrap a .row class around your .col- classes, then wrap the .grid class around your .row classes and at last, use the .container as a wrapper for your .grid. Like mentioned earlier, you can add as many .row classes as you'd like with as many different .col- classes you'd like to have.
